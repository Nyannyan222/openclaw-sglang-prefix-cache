# WildClawBench Phase 2 Roadmap

## Objectives

Phase 2 turns the Search Retrieval pilot into a broader experiment:

1. Expand to more WildClawBench categories.
2. Add formal manual and LLM-judge grading.
3. Replay prompts through SGLang runtime to measure cached tokens, prefill
   tokens, and latency.
4. Prepare the implementation path for runtime-level semantic sub-context KV
   reuse.

## Formal Grading

The grading rubric is stored in:

```text

```

Dimensions:

- `task_correctness`, weight 0.35
- `evidence_sufficiency`, weight 0.20
- `context_grounding`, weight 0.15
- `output_format_compliance`, weight 0.10
- `safety_compliance`, weight 0.15
- `efficiency_observation`, weight 0.05

The manual grading sheet is seeded from the 12-row combined Search Retrieval
run so human review can be added without changing the pipeline.

## SGLang Runtime Replay

Runtime replay script:

```bash
python scripts/run_wildclaw_sglang_runtime_replay.py \
  --manifest benchmark_results/wildclaw_phase2/wildclaw_phase2_sglang_runtime_manifest.jsonl \
  --base-url http://127.0.0.1:8000/v1 \
  --model <served-model-name> \
  --repeat 2
```

It records:

- response usage tokens,
- estimated cached tokens,
- estimated prefill tokens,
- request latency,
- selected SGLang Prometheus metric deltas.

This gives a clean runtime baseline for ordinary prefix-cache reuse.

## Semantic KV Reuse Path

The actual runtime-level semantic sub-context KV reuse is still a separate
implementation step. The proposed path is:

1. Canonicalize each semantic sub-context block.
2. Assign stable hashes to each block.
3. Add block-level metadata to runtime prompts.
4. Patch SGLang cache lookup to recognize reusable non-prefix semantic blocks.
5. Compare three modes:
   - no cache reuse,
   - standard prefix cache,
   - semantic sub-context KV reuse.
6. Report cached tokens, prefill tokens, TTFT, and latency.

The current replay manifest is deliberately compatible with all three modes so
we can keep task data fixed while changing only runtime behavior.
